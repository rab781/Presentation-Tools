## 2024-03-22 - Optimize VoiceRecognizer stream initialization
**Learning:** Initializing the audio input stream (`with self.microphone as source:`) inside a continuous listening loop carries significant performance overhead, especially because the underlying C libraries (like PyAudio) must acquire hardware handles, allocate buffers, and drop them constantly. We discovered `_listen_loop` was re-opening the mic stream on every single `sr.WaitTimeoutError` (i.e., every second of silence).
**Action:** Use a nested loop structure for continuous listening tasks. Enter the microphone context manager in the outer loop, and perform continuous read/listen operations inside the inner loop. Catch timeouts within the inner loop to keep the stream alive, but catch other critical exceptions to break out and re-initialize the stream via the outer loop.
